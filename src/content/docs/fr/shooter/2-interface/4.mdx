Il existe trois fonctions très utiles pour créer des interfaces.

- Pour afficher du texte ou le contenu d'une variable :

```lua
print(texte, x, y, couleur)
-- exemple :
print("mon texte", 90, 2, 6)
```

- Pour afficher les contours d'un rectangle :

```lua
rect(x1, y1, x2, y2, couleur)
-- exemple :
rect(20, 20, 108, 108, 9)
```

:::note[Coordonnées du rectangle]
`x1` `y1` représente le coin haut-gauche du rectangle, et `x2` `y2` le coin bas-droite.
:::

- Pour afficher un rectangle rempli avec la couleur :

```lua
rectfill(x1, y1, x2, y2, couleur)
```

En utilisant ces trois fonctions, il est possible de composer des interfaces complètes.

## 2. Ajouter une interaction au game over

Nous devrions permettre au joueur de recommencer le jeu en appuyant sur une touche. Nous n'allons pas utiliser la touche <kbd>X</kbd>, car il est possible que le joueur soit déjà en train d'enfoncer la touche parce qu'il tirait pendant la partie.

Dans PICO-8, il y a 2 touches d'action : <kbd>X</kbd> et <kbd>C</kbd>, qui sont représentées par les symboles X et O dans le code. Nous utiliserons donc la touche <kbd>C</kbd> (symbole O).

Ecrivez la condition suivante dans `update_gameover()`. Vous pouvez écrire le symbole O avec <kbd>Shift</kbd> <kbd>O</kbd>.

<img src="/images/goupd.png" />

:::note[pourquoi appeler init ?]
Au lieu d'appeler `_init()` dans la condition, vous auriez peut-être voulu écrire `etat = 0` pour repasser à l'**état jeu**. Le problème, c'est que vous reviendriez à la scène de jeu telle qu'elle était juste avant, c'est-à-dire avec votre vaisseau en contact avec l'ennemi.

En appelant `_init()`, vous redonnez aux variables leurs valeurs de départ, ce qui remet `etat` à `0` par la même occasion.
:::

Pour finir, vous pouvez ajouter le texte « Appuyez sur C pour continuer » sur votre interface.
